Frequently Asked Questions

What type of data does this site analzye? Spotify listening history

Spotify allows you to request and download your past listening history. This comes in the form of a number of JSON files which list every time you have stopped and started a song, along with other data. Spotify has instructions on how to request this data here. The site also comes with demo data availble to try it out, and examples of what types of analysis is possible here.

Does my data get sent anywhere while the site is analzying it? No! Your data never leaves your computer

All analysis of the data is done locally on your own computer. Analysis is done using code written in typescript, meaning it runs directly in your browser. Your data doesn't get sent to a server, and no copy of it is ever made or stored anywhere but your computer.

How does is it determined if I "skip" a song? If you listen to a song for less than 5 seconds

Spotify data records both the time you start a song, and the time you stop listening to a song. If those times are within 5 seconds of each other it is counted as a skip. Currently this is an arbitrarely chosen metric. In the future the plan is to include a slider which allows a user to select the threshold for a skip.

What is the SkipsPerPlay metric? The ratio of how often you skip a song vs play it

This metric stems from a personal need. I found that I had a lot of songs in my library that I didn't actually enjoy listening to. I figured this metric would be an easy way to filter out which songs I enjoyed listening to, versus those I didn't. The higher the ratio, the more often I skip a song and the less I like it.
